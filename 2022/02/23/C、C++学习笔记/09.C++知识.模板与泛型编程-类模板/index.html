类模板

类模板，也是产生类的模具，通过给定的模板参数生成具体的类，也就是实例化一个特定的类，这个概念听起来和函数模板差不多。

实例化类模板： myvector<int> tmpvec; 编译器就会生成一个具体的类(类名为myvector)，类模板参数中的T就被替换成了通过”<>“传进来的具体的类型，这里就是int类型。

myvector是类模板名，不是一个类型名(或者说是一个残缺的类型名)，类模板是用来实例化类的。

ps：在main.cpp源文件中导入myvector.h头文件，并在main函数中添加代码myvector<int> tmpvec ，编译链接环节发现会报错，那是因为myvector没有写构造函数。

类模板的成员函数

类模板的成员函数可以写在.h文件的类模板定义中，这种写在类模板定义中的成员函数就被隐式声明为内联函数。

当然，类模板的成员函数也可以在vector类模板定义中声明，而在类模板定义之外实现。

注意：一个类模板虽然里面可能有很多成员函数，但是，当实例化模板之后，如果后续没有使用某个成员函数，则这个成员函数是不会被实例化的。也就是，一个实例化的模板，它的成员只有在使用的时候才会被实例化。

总结：同一个函数模板，根据传递进去的参数不同，就能够推断出不同的类型。

模板显示实例化与声明

模板只有被使用时才会被实例化。

但是有个问题，一个项目中可能有多个.cpp源码文件(也可能是扩展名的源码文件)。

现在要编译这个项目，已经知道的事实是；这些.cpp源文件对于编译器来讲都是独立编译的(每个.cpp编译后可能生成一个.obj文件)。

这两个.cpp代码中的”A<float> a(1,2);“这行代码在编译的时候，因为每个.cpp文件独立编译，所以编译器在myproj.cpp中实例化出一个A<float>类(也可以叫模板类A的一个实例)，在test.cpp中也会实例化出一个A<float>类，可想而知，多个.cpp都实例化出来了相同的类模板，如果项目很大，.cpp文件很多，那这个额外的开销比较大，增加了很多编译时间并且没有必要。

可以通过”显式实例化“来避免这种生成很多个相同类模板实例的开销。

可以在test.cpp头文件写入如下代码

templata A<float>; 这叫”实例化定义“，只有一个.cpp文件里这样写，编译器为其生成代码

然后，在其他的.cpp文件中，当然不需要再实例化了，只需要在其他的.cpp的头上声明这个实例化出来的类就行了，可以这样写:

extern template A<float>;

这个带extern的代码行就被称为”模板实例化声明“，当编译器遇到extern模板实例化声明时，就不会在本.cpp源文件中生成一个extern后面所表示的类模板的实例化版本代码。

函数模板 也是一样的。

函数模板实例化定义 template void myfunc(int& v1, int& v2);

函数模板实例化声明 extern template void myfunc(int& v1, int& v2);

注意：模板的实例化定义只有一个，模板的实例化声明可以有多个。
