拷贝赋值运算符(赋值运算符)

如果给对象赋值，那么系统会调用一个拷贝赋值运算符(简称赋值运算符)

如果没有自己书写拷贝赋值运算符重载，编译器会用默认的对象赋值规则为对象赋值，甚至在必须要

的情况下帮助我们重载赋值运算符。当然编译器生成的重载赋值运算符功能上比较粗糙，只能完成

一些简单的赋值以及调用类类型成员变量所对应类中提供的拷贝赋值运算符。

但为了精确地控制类对象的赋值动作，往往会由程序员自己来重载赋值运算符

如果想要禁止Time类型对象之间的赋值，可以在.h头文件声明赋值运算符时，用private修饰即可。

析构函数(释放函数)

析构函数与构造函数正好是相对的，或者说是相反的，定义对象的时候，系统会调用构造函数，对象销毁的时候，系统会调用析构函数。 析构函数也没有返回值。

如果不写自己的析构函数，编译器可能会生成一个”默认析构函数”,也可能不会生成一个”默认析构函数”，是否生成取决于具体需要

子类

从父类派生出子类。有父类，有子类，这种层次关系就叫”继承(也叫继承性)”。

父类中主要是定义一些公共的成员变量和成员函数，然后通过继承这个父类来构建新的类

这个新类称为子类，通过继承父类很多成员变量、成员函数就自动继承到子类中去了。

子类一般比父类更加庞大，占用的内存空间比父类多

访问等级(public、protected、private)

三种访问权限 三种继承方式
public: 可以被任意实体所访问 public继承
protected:只允许本类或者子类的成员函数访问 protected继承
private:只允许本类的成员函数访问 private继承
父类中的访问权限 子类继承父类的继承方式 子类得到的访问权限
public public public
protected public protected
private public 子类无权访问
public protected protected
protected protected protected
private protected 子类无权访问
public private private
protected private private
private private 子类无权访问

函数遮蔽

正常情况下，父类中的成员函数只要是用public或者protected修饰的，

子类只要不用private继承方式来继承父类，那么子类中都可以调用。

但是，在C++的类继承中，子类会遮蔽父类中的同名函数，不论此函数的返回值、参数。

也就是说，父类和子类中的函数只要名字相同，子类中的函数就会遮蔽掉父类中的同名函数。

虚函数

★★★这一节很重要★★★

防止在子类中写错虚函数，C++11中，可以在虚函数声明所在行的末尾增加一个override关键字

override这个关键字是用在子类中的，而且是虚函数专用的。

与override关键字相对的还有一个final关键字，final关键字也是用于虚函数和父类中的。

如果在函数声明的末尾增加filan，那么任何在子类中尝试覆盖该成员函数的操作都将引发错误。

另外，子类的虚函数返回类型一般也和父类所要覆盖的虚函数返回类型一样，也可以有点小差别

动态绑定：在程序运行的时候才能知道调用了哪个子类的函数。

一个函数如果不去调用，编码的时候可以只写该函数的声明部分，不写该函数的定义部分。

但是虚函数，因为是在程序运行的时候才知道调用了哪个虚函数，所以

虚函数必须写它的定义部分（以备编译器随时使用随时就存在），否则会编译出错。
