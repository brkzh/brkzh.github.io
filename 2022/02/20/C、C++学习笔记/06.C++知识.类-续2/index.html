RTTI、dynamic_cast、typeid、type-info与虚函数表

RTTI

RTTI(Run Time Type Identification),运行时类型识别

RTTI 可以看作系统提供出来的一种功能，或者说是一种能力，需要通过两个运算符来实现

上述两个运算符要能够正常的如所期望的那样工作，父类中至少要有一个虚函数，不然这两个运算符工作的结果很可能与预期的不一样。

因为只有虚函数的存在，这两个运算符才会使用指针或者引用所指对象的类型(new时的类型)。

dynamic_cast运算符

dynamic_cast运算符能将父类指针或者引用安全地转换为子类的指针或者引用

注意： 使用dynamic_cast运算符的前提条件是：父类中必须至少有一个虚函数。

针对引用类型，dynamic_cast运算符使用

引用类型 判断类型转换是否成功和 指针类型 不太一样，指针类型是判断是否为空指针，而引用是没有空引用这个说法的。

所以，对于引用转换失败这种情况，程序会抛出要给 std::bad_cast 异常，标准库头文件有定义，捕捉异常常用 tyr{} 和 catch(){}。

RTTI与虚函数表

在C++中，如果这个类里面要含有虚函数，编译器就会针对该类产生一个虚函数表，虚函数表里面有很多表项，每一项都是一个指针，每个指针指向这个类里面的各个虚函数的入口地址。

基类与派生类关系详细再探讨

派生类、派生类构造函数等

一个派生类对象，其实是包含多个组成部分(多个子对象)的

构造一个派生类对象时，基类的构造函数会被调用，派生类的构造函数也会被调用。

派生类实际上是使用基类的构造函数来初始化它的基类部分。

基类控制基类部分的成员初始化，派生类控制派生类部分的成员初始化。

构造派生类对象时，基类的构造函数需要参数，则就需要通过**派生类构造函数的初始化列表**来给基类的构造函数传递参数。

左值、右值、左值引用、右值引用

左值和右值

**左值 **，能用在赋值语句等号左侧的内容(它得代表一个地址)。

**右值 **，不能作为左值的值。

c++中的一条表达式，要么就是右值，要么就是左值，不可能两者都是。

但是一个左值有时候又能被当作右值使用。如：i = i + 1;

一个左值它可能同时具有左值属性和右值属性。

左值表达式就是左值，右值表达式就是右值。

引用分类

左值引用

左值引用就是引用左值的，就是绑定到左值的引用。

右值引用

右值引用就是引用右值的，就是绑定到右值的引用。

右值引用就是必须绑定到右值的引用，要通过 “&&” 而不是 “&”来获得右值引用。右值引用也是引用。

左值引用绑左值，右值引用绑右值，两者正好相反，能绑定左值引用的内容一般绑不到右值引用上去，反之亦然。

返回左值引用的函数，连同赋值、下标、解引用和前置递增递减运算符等，都是返回左值表达式(左值)的例子，可以将一个左值引用绑定到这类表达式的结果上。

返回非引用类型的函数，连同算数、关系、位以及后置递增运算符，都生成右值，不能将一个左值引用绑定到这类表达式上，但可以将一个const的左值引用或者一个右值引用绑定到这类表达式的结果上。

额外解释前置递增递减运算符和后置递增递减运算符。

前置递增递减运算符，如++i，–i，是先 加/减 后用的意思。

++i是直接给i变量加1，然后返回i本身，因为i是变量，所以可以被赋值，因此是左值表达式

后置递增递减运算符，如i++，i–，是 先用 后 加/减的意思。

i++先产生一个临时变量来保存i的值用于使用目的，再给i加1，接着返回临时变量，之后系统再释放这个临时变量，临时变量被释放掉了，不能再被赋值，因此是右值表达式。
