智能指针 shared_ptr

引言

在函数内定义一个变量(对象)，那么一旦离开这个函数，这个变量的声明周期就结束了。而局部静态变量(static变量/对象)是在第一次使用前分配内存，这个变量的内存即使离开这个函数也不释放，一直到程序结束的时候才释放。

上面谈的 变量(对象)一般都是在栈或者静态内存上分配和释放变量所需要的空间，不需要程序员去干涉。

对象也可以new出来，new一个对象实际上是在堆上分配内存。而自己new出来的对象必须自己用delete释放。

由程序员自己用new出来的对象分配内存（new返回的是该对象的指针）的方式叫做动态分配。这种动态分配的内存的方式就相当于对内存直接进行管理，具体地说就是针对new和delete的应用。

C++11中，auto可以和new一起使用

const对象也可以动态分配内存

new/delete探秘

new/delete是什么

new和delete都是关键字(也叫运算符/操作符)，都不是函数。

new/delete这一对比malloc/free这一对多做了不少事情，例如，new相比于malloc不但分配了内存，还会额外做一些初始化工作，而delete相比于free不但释放了内存，还会额外做了一些清理工作。

既然后调用构造函数和析构函数的能力，这就意味着new和delete具备针对堆所分配的内存进行初始化(把初始化代码放在类的构造函数)和释放(把释放相关的代码放在类的析构函数中)的能力，而这些能力是malloc和free所不具备的。

operator new()和operator delete()

重载运算符实际的表现就是一个函数，所以operator new(...) 和 operator delete(...) 实际上是函数。

那么这两个函数和new/delete操作符有什么关系呢？

为什么new/delete、new[]/delete[]要配对使用

代码行 int* p = new int[2]; 并没有多分配出4字节内存，释放的时候 delete p; 也能正确释放。

这种内置类型如int类型，double类型，delete的时候不存在调用类的析构函数的说法(只有类对象或者类对象数组delete的时候才会存在调用析构函数的说法)，所以new的时候系统并没有多分配4字节内存，系统在调用operator delete(…)函数来释放内存时，因为new内部还有一些其他深入的工作原理，系统能够知道要释放的内存带线啊哦究竟是多少。

智能指针总述

直接new一个对象的方式返回的时一个对象的指针，这个对象的指针很多人称其为裸指针。

智能指针可以理解成对“裸指针”进行了包装。最突出的就是智能指针能够“自动释放所指向的对象”。

在c++11中，已经抛弃了 std::auto_ptr 这种智能指针。

std::unique_ptr, std::shared_ptr, std::weak_ptr　这三种智能指针在c++11中推荐使用。

这三种智能指针其实都是类模板，可以将new获得的地址赋给它们。

shared_ptr基础

shared_ptr指针采用的是共享所有权来管理所指向对象的生存期。所以，对象不仅仅能被一个特定的shared_ptr所拥有，而是能够被多个shared_ptr所拥有。

在决定是否要采用这种指针时首先得想一个问题：所指向的对象是否需要被共享，如果需要被共享(类似于多个指针都需要指向同一块内存)，那就是用shared_ptr，如果只需要独享(只有一个指针指向这块内存)，就建议使用unique_ptr智能指针。

shared_ptr的工作机制是使用引用计数，每一个shared_ptr指向相同的对象(内存)，所以只有最后一个指向该对象的shared_ptr指针不需要再指向该对象时，这个shared_ptr才回去析构所指向的对象。

最后要给指向该对象的shared_ptr在什么情况下会释放该对象呢？

shared_ptr的 一般形式：shared_ptr<指向的类型> 智能指针名;

shared_ptr<string> sps; 一个指向string的智能指针，名字为sps

这种默认初始化的情形，该智能指针里面保存的是一个空指针nulltr(可以指向类型为string的对象)。

对于返回值为shared_ptr类型

make_shared函数

这个一个标准库里的函数模板，被认为是最安全和更高效的分配和使用shared_ptr智能指针的一个函数模板。

它能够在动态内存(堆)中分配并初始化一个对象，然后返回指向此对象的shared_ptr。

make_shared使用起来还行，但是如果自定义删除器，那么使用make_shared方法生成的shared_ptr对象，就没有版本自定义删除器了。

shared_ptr指针常用操作

1.use_count成员函数

该成员函数用于返回多少个智能指针指向某个对象。

2.unique成员函数

是否该智能指针独占某个指向的对象，也就是若只有一个指针指针指向某个对象，则unique返回true，否则返回false

3.reset成员函数

(1)当reset不带参数时

若pi是唯一指向该对象的指针，则释放pi指向的对象，将pi置空。

若pi不是唯一指向该对象的指针，则不释放pi指向的对象，但指向该对象引用计数会减1，同时将pi置空。

(2)当reset带参数(一般是一个new出来的指针)时

若pi是唯一指向该对象的指针，则释放pi指向的对象，让pi指向新内存。

若pi不是唯一指向该对象的指针，则不释放pi指向对象，但指向该对象的引用计数会减1，同时让pi指向新内存。

(3)空指针也可以通过reset来重新初始化

4. *解引用

*p：解引用的感觉，获得p指向的对象。

5.get成员函数

p.get()：返回p中保存的指针。小心使用，若智能指针释放了所指向的对象，则返回的这个所指向的对象也就变的无效了。

为啥要有这个函数？因为有些函数的参数需要的是一个内置指针(裸指针)，所以需要通过get取得这个内置指针并传递给这样的函数。

但是注意，不要delete这个get到的指针，否则会产生不可预料的后果。

6.swap成员函数

用于交换两个智能指针所指向的对象。引用计数并不会发生变化

7. =nullptr

· 将所指向对象的引用计数减1，若引用计数变为0，则释放智能指针所指向的对象

· 将智能指针置空

9.指定删除器和数组问题

1)指定删除器

智能指针在一定的时机自动删除它所指向的对象。默认情况下，shared_ptr正是使用delete运算符作为默认的删除它所指向的对象的方式。

同时，程序员可以指定自己的删除器，这样当智能指针需要删除一个它所指向的对象时，它不再去调用默认的delete运算符来删除对象，而是调用程序员为它提供的删除器来删除它所指向的对象。

shared_ptr指定删除器的方法比较简单，一般只需要在参数中添加具体的删除器函数即可（注意，删除器是一个单形参的函数）。

删除器也可以是一个lambda表达式，lambda可以理解为一个匿名函数(是一个表达式，但使用起来也比较像一个匿名函数)。注意，用{}包着的部分就是lambda表达式组成部分，直接作为一个参数来用：

常规情况下，默认的删除器工作是挺好，但是有一些情况需要自己指定删除器，默认的删除器处理不了。

用shared_ptr管理动态数组的时候，就需要指定自己的删除器，默认的删除器不支持数组对象。

如果一个类定义中带有析构函数，那么程序员必须指定自己的删除器，否则会报异常。

此外还可以将default_delete作为删除器，这是一个标准库里的类模板，注意写法，这个删除器的内部也是通过delete来实现功能的：

shared_ptr<A>pA(new A[10], std::default_delete<A[]>());

2)指定删除器额外说明

就算是两个shared_ptr指定的删除器不同，只要他们所指向的对象相同，那么这两个shared_ptr也属于同一个类型。

同一个类型有个明显的好处，就是可以放到元素类型为该对象类型的容器里，方便操作

vector<shared_ptr<int>> pvec{p1, p2};

make_shared是被提倡的生成shared_ptr的方法，但是如果用make_shared方法生成shared_ptr对象，那就没有办法自定义删除器了。
